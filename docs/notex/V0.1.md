## 第 0 步：了解 Tauri

Tauri 是一个用于为所有主流桌面和移动平台构建微小、快速二进制文件的框架。开发者可以使用任何编译为 HTML、JavaScript 和 CSS 的前端框架来构建用户界面，并在需要时利用 Rust、Swift 和 Kotlin 等语言来处理后端逻辑。

优势：安全、应用体积小、架构灵活

### 安全基石
- 由于构建于 Rust 之上，Tauri 能够充分利用 Rust 提供的内存安全、线程安全和类型安全特性。即使开发者不是 Rust 专家，使用 Tauri 构建的应用也能自动获得这些优势。
- Tauri 还会对其主要和次要版本进行安全审计。这不仅涵盖了 Tauri 组织内部的代码，还包括 Tauri 所依赖的上游依赖项。（这并不能消除所有风险，但它为开发者提供了一个坚实的基础。）

### 应用体积小
Tauri 应用利用每个用户系统上已有的 webview。
- Tauri 应用仅包含该应用特定的代码和资源，无需为每个应用捆绑浏览器引擎。
- 极简的 Tauri 应用体积可以小于 600KB。

### 架构灵活

由于 Tauri 使用 Web 技术，这意味着几乎任何前端框架都与 Tauri 兼容。
- 开发者可以使用 JavaScript 中的 invoke 函数实现 JavaScript 与 Rust 之间的绑定，同时 Tauri 插件也支持 Swift 和 Kotlin 绑定。
- TAO 负责 Tauri 窗口创建，而 WRY 负责 webview 渲染。这些库由 Tauri 维护，如果需要超出 Tauri 所暴露功能的深度系统集成，可以直接使用这些库。
- 此外，Tauri 维护了许多插件来扩展核心功能。


**前端与后端的关系**：

```
前端（WebView）  ─── invoke() ──→  Rust 后端（系统进程）
   UI 渲染         IPC 通信          系统调用 + 安全控制
```

- 前端运行在 WebView 中，负责 UI 渲染
- Rust 运行在系统进程中，负责文件/数据库/系统 API
- 通过 `invoke()` 通信（前端调 Rust 命令）

## 第 2 步：创建项目

Tauri 能够与几乎任何前端框架配合使用，项目架构非常灵活，官方提供了 `create-tauri-app` 工具，以便使用官方维护的框架模板创建新的项目。

- 目前包含用于原生（无框架的 HTML、CSS 和 JavaScript）、Vue.js、Svelte、React、SolidJS、Angular、Preact、Yew、Leptos 和 Sycamore 的模板。
- 还可以在 Awesome Tauri 仓库 中找到或添加其他社区模板和框架。

编译默认生成 2 个安装包（`msi` 标准化安装，适合于无人值守情况；`exe` 自由度高）

- .\src-tauri\target\release\bundle\msi\notex_0.1.0_x64_en-US.msi
- .\src-tauri\target\release\bundle\nsis\notex_0.1.0_x64-setup.exe

## 第 3 步：了解项目结构

Tauri 项目由**前端**和 **Rust 后端**两部分组成：

```
notex/
├── index.html              ← 前端入口（Vite 要求在根目录）
├── package.json            ← npm 配置
├── pnpm-lock.yaml          ← pnpm 锁文件（锁定依赖的确切版本）
├── tsconfig.json           ← TypeScript 配置
├── vite.config.ts          ← Vite 构建配置
├── src/                    ← 前端代码
│   ├── main.ts             ← 前端入口脚本
│   └── styles.css          ← 样式
└── src-tauri/              ← Rust 后端
    ├── build.rs            ← Tauri 构建脚本
    ├── Cargo.toml          ← Rust 依赖
    ├── tauri.conf.json     ← Tauri 应用配置（主配置文件）
    ├── capabilities/       ← 权限声明
    │   └── default.json
    ├── icons/              ← 应用图标
    └── src/
        ├── main.rs         ← Rust 二进制入口
        └── lib.rs          ← Tauri 配置入口
```

## 第 5 步：修改 Rust 后端

**src-tauri/build.rs**：
```rust
fn main() {
    tauri_build::build()
}
```

**src-tauri/src/lib.rs**：
```rust
use tauri::Builder;
pub fn run() {
    Builder::default().run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

**src-tauri/capabilities/default.json**：
```json
{
  "$schema": "../gen/schemas/desktop-schema.json",
  "identifier": "default",
  "description": "NoteX 默认权限",
  "windows": ["main"],
  "permissions": [
    "core:default",
    "opener:default"
  ]
}
```
