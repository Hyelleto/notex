# V0.2 — 笔记列表 + 编辑区 UI

**目标**：搭建前端界面骨架，实现笔记列表、编辑区和状态栏。

**TDD**：❌ 前端 UI 不做 TDD。

> **为什么不做 TDD？**
>
> TDD 适合有明确输入输出的逻辑（函数、命令）。前端 UI 的验证标准是"看起来对不对"、"交互顺不顺畅"，这类判断难以用代码断言。
>
> **替代方案**：用户故事充当测试规范。每个用户故事的验收标准就是测试用例，只是执行方式是手动操作而非 `cargo test`。
>
> | 层次 | 测试方式 | 本版本 |
> |------|----------|--------|
> | Rust 逻辑 | `cargo test` 自动化 | 无（无 Rust 代码） |
> | 前端逻辑 | vitest 等单测框架 | 暂不做（后续版本引入） |
> | UI 交互 | 手动验证 | ✅ 本版本使用 |

**前置条件**：V0.1 已完成（`npm run tauri dev` 能弹出 Hello World 窗口）。

## 第 1 步：定义类型

**src/types.ts**：

```ts
export interface Note {
  id: number;
  title: string;
  content: string;
}
```

- `interface`：TypeScript 接口，描述对象的结构
- `export`：导出供其他文件导入
- `id`：笔记唯一标识
- `title`：笔记标题（取内容第一行）
- `content`：笔记正文

## 第 2 步：编写前端逻辑

**src/main.ts**：

### 2.1 导入与 DOM 引用

```ts
import type { Note } from "./types";

const editor = document.getElementById("editor") as HTMLTextAreaElement;
const charCount = document.getElementById("char-count")!;
const lineCount = document.getElementById("line-count")!;
const noteList = document.getElementById("note-list")!;
const btnNew = document.getElementById("btn-new")!;
```

- `import type`：类型导入，编译后移除，不影响运行时
- `as HTMLTextAreaElement`：类型断言，告诉 TS 这是 textarea 元素（可以访问 `.value`）
- `!`：非空断言，告诉 TS 这个元素一定存在（不会是 null）

### 2.2 应用状态

```ts
let notes: Note[] = [
  { id: 1, title: "欢迎使用 NoteX", content: "# 欢迎使用 NoteX\n\n这是一个简单的记事本应用。" },
];
let activeNoteId = 1;
```

- `notes`：笔记数组，前端的本地数据源
- `activeNoteId`：当前选中的笔记 ID
- 默认有一篇欢迎笔记

### 2.3 状态栏更新

```ts
function updateStats() {
  charCount.textContent = `${editor.value.length} 字`;
  lineCount.textContent = `${editor.value.split("\n").length} 行`;
}
```

- `` `${}` ``：模板字符串，嵌入变量
- `editor.value`：textarea 的当前内容
- `split("\n")`：按换行符分割为数组

### 2.4 渲染笔记列表

```ts
function renderNoteList() {
  noteList.innerHTML = "";
  for (const note of notes) {
    const div = document.createElement("div");
    div.className = `note-item${note.id === activeNoteId ? " active" : ""}`;
    div.textContent = note.title;
    div.addEventListener("click", () => {
      activeNoteId = note.id;
      editor.value = note.content;
      updateStats();
      renderNoteList();
    });
    noteList.appendChild(div);
  }
}
```

- `innerHTML = ""`：清空列表（全量重渲染）
- `createElement`：创建 DOM 元素
- `dataset.id`：`data-id` 属性（后续版本使用）
- `addEventListener("click", ...)`：点击事件
- `appendChild`：插入到 DOM

### 2.5 新建笔记

```ts
function createNote() {
  const id = Date.now();
  notes.push({ id, title: "新笔记", content: "" });
  activeNoteId = id;
  editor.value = "";
  updateStats();
  renderNoteList();
}
```

- `Date.now()`：时间戳作为临时 ID（后续迭代改为数据库自增 ID）
- `push`：添加到数组末尾

### 2.6 编辑事件

```ts
editor.addEventListener("input", () => {
  const note = notes.find((n) => n.id === activeNoteId);
  if (note) {
    note.content = editor.value;
    note.title = editor.value.split("\n")[0].replace(/^#+\s*/, "").trim() || "新笔记";
    renderNoteList();
  }
  updateStats();
});
```

- `input` 事件：textarea 内容变化时触发（比 `change` 更实时）
- `find`：查找匹配元素
- `.replace(/^#+\s*/, "")`：正则去除 Markdown 标题符号 `#`
- `|| "新笔记"`：标题为空时的默认值

### 2.7 初始化

```ts
btnNew.addEventListener("click", createNote);
renderNoteList();
updateStats();
```

## 第 3 步：更新 HTML

**index.html**：

```html
<body>
  <div class="app">
    <header class="toolbar">
      <button id="btn-new">+ 新建</button>
      <span class="title">NoteX</span>
    </header>
    <main class="content">
      <aside class="sidebar" id="note-list"></aside>
      <section class="editor-area">
        <textarea id="editor" placeholder="开始写下你的笔记..."></textarea>
      </section>
    </main>
    <footer class="statusbar">
      <span id="char-count">0 字</span>
      <span id="line-count">0 行</span>
    </footer>
  </div>
  <script type="module" src="/src/main.ts"></script>
</body>
```

- `<header>` / `<main>` / `<footer>` / `<aside>` / `<section>`：HTML5 语义化标签
- `id` 属性：JS 通过 `getElementById` 获取元素
- `placeholder`：textarea 占位提示文字
- `id="note-list"` 为空，由 JS 动态填充

## 第 4 步：编写样式

**src/styles.css**：

### 4.1 全局重置

```css
* { margin: 0; padding: 0; box-sizing: border-box; }
html, body { height: 100%; background: #1a1a2e; color: #e0e0e0;
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "Microsoft YaHei", sans-serif; }
```

- `*`：通配选择器，重置所有元素的默认边距
- `box-sizing: border-box`：宽高包含 padding 和 border
- `#1a1a2e`：深蓝背景色
- `#e94560`：强调色（红色）

### 4.2 三栏布局

```css
.app { height: 100vh; display: flex; flex-direction: column; }
.content { flex: 1; display: flex; overflow: hidden; }
.sidebar { width: 200px; ... }
.editor-area { flex: 1; display: flex; }
```

```
┌─────────────────────────────────────┐
│ .toolbar（固定 40px）                │
├──────────┬──────────────────────────┤
│ .sidebar │ .editor-area             │
│ (200px)  │ (flex:1 占满剩余)         │
├──────────┴──────────────────────────┤
│ .statusbar（固定 24px）              │
└─────────────────────────────────────┘
```

- `display: flex`：弹性布局
- `flex-direction: column`：垂直排列
- `flex: 1`：占满剩余空间
- `overflow: hidden`：内容溢出时隐藏

### 4.3 笔记列表项

```css
.note-item { padding: 8px 16px; cursor: pointer; font-size: 13px;
  white-space: nowrap; overflow: hidden; text-overflow: ellipsis;
  border-left: 3px solid transparent; }
.note-item:hover { background: rgba(233, 69, 96, 0.1); }
.note-item.active { background: rgba(233, 69, 96, 0.15); border-left-color: #e94560; color: #fff; }
```

- `white-space: nowrap`：不换行
- `text-overflow: ellipsis`：溢出显示省略号
- `.active`：当前选中项的高亮样式

### 4.4 编辑区

```css
#editor { flex: 1; border: none; padding: 20px 24px;
  font-family: "Fira Code", "Consolas", monospace; font-size: 14px; line-height: 1.6;
  resize: none; outline: none; background: #1a1a2e; color: #c9d1d9; }
```

- `resize: none`：禁止手动调整大小
- `outline: none`：去除聚焦边框
- 等宽字体：适合代码/Markdown 编辑
