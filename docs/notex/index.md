# NoteX — Tauri + Rust 实战教程

> 基于 Tauri v2 | TDD 驱动 | 从用户故事出发

## 教学理念

**用户故事 → 集成测试 → 单元测试 → 实现 → 验证**

不从"写什么代码"开始，而从"用户要什么"开始。

## TDD 核心流程

```
用户故事 → 集成测试 → 单元测试 → 实现 → 验证
```

| 步骤 | 说明 |
|------|------|
| 用户故事 | "作为用户，我想 XXX" |
| 集成测试 | 模拟用户完整操作流程 |
| 单元测试 | 拆解为独立函数的测试 |
| 实现 | 写最小代码让测试通过 |
| 验证 | 所有测试通过 + 功能可用 |
