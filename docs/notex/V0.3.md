# V0.3 — 前后端打通 ⭐ TDD

**目标**：前端通过 `invoke` 调用 Rust 命令，笔记数据在 Rust 侧存储。

**TDD**：✅ 用户故事 → 集成测试 → 单元测试 → 实现。

**前置条件**：V0.2 已完成（笔记列表 UI 可用）。

---

## 用户故事

| ID | 用户故事 | 验收标准 |
|----|----------|----------|
| US5 | 编辑笔记后切回来，内容还在 | 编辑 → 切换 → 切回 → 内容一致 |
| US6 | 新建笔记立即显示在列表 | 点击新建 → 列表出现新项 |
| US7 | 多篇笔记互不干扰 | 编辑 A → 切到 B → 切回 A → A 内容不变 |
| US8 | 首次打开有默认笔记 | 空应用启动 → 显示欢迎笔记 |
| US9 | 连续新建多篇笔记 | 连续点新建 → 列表逐个增加 |
| US10 | 加载不存在的笔记有提示 | 调用 load_note(999) → 返回错误 |

## TDD 流程说明

本版本是第一个使用 TDD 的版本。流程如下：

```
第 2 步：写集成测试（用户故事）  → 🔴 编译失败（类型和函数不存在）
第 3 步：写单元测试 + 实现        → 🟢 12/12 通过
第 4 步：连接前端                  → 功能可用
```

**为什么要先写测试？**
- 集成测试定义了"用户会怎么用"，而非"函数应该怎么写"
- 先看到失败（红），再写最小实现让测试通过（绿）
- 测试驱动出组件需求，而非反过来

## 第 2 步：写集成测试（RED）

> **核心思想**：先写测试，不写实现。测试编译失败，证明测试在起作用。

在 `src-tauri/src/lib.rs` 中，先写集成测试。此时 `Note`、`NoteStore`、`store_save` 等都不存在，测试必然编译失败。

### 2.1 命令层包装

```rust
#[cfg(test)]
mod tests {
    use super::*;

    fn cmd_save(store: &NoteStore, id: u32, title: String, content: String) -> Result<(), String> {
        let mut map = store.lock().map_err(|e| e.to_string())?;
        store_save(&mut map, Note { id, title, content });
        Ok(())
    }
    fn cmd_load(store: &NoteStore, id: u32) -> Result<Note, String> {
        let map = store.lock().map_err(|e| e.to_string())?;
        store_load(&map, id).ok_or_else(|| format!("笔记 {} 不存在", id))
    }
    fn cmd_list(store: &NoteStore) -> Result<Vec<Note>, String> {
        let map = store.lock().map_err(|e| e.to_string())?;
        Ok(store_list(&map))
    }
```

**为什么需要 cmd_save/cmd_load？**

Tauri 命令（`#[tauri::command]`）依赖 `tauri::State` 注入，无法在测试中直接调用。`cmd_*` 函数包装了 `NoteStore`，绕过 Tauri 依赖，可独立测试完整命令流程。

```
测试中：cmd_save(&store, ...)     → 不需要 Tauri 运行时
运行时：save_note(tauri::State)   → Tauri 自动注入 State
```

### 2.2 用户故事测试

```rust
    // US5：编辑后切回来内容还在
    #[test]
    fn us5_edit_and_switch_back() {
        let store = NoteStore::default();
        cmd_save(&store, 1, "新笔记".into(), "".into()).unwrap();
        cmd_save(&store, 1, "Hello".into(), "Hello World".into()).unwrap();
        let note = cmd_load(&store, 1).unwrap();
        assert_eq!(note.content, "Hello World");
    }
```

- 模拟：新建笔记 → 编辑内容 → 重新加载 → 内容一致
- `unwrap()`：测试中直接解包，失败则 panic

```rust

    fn us7_notes_independent() {
        let store = NoteStore::default();
        cmd_save(&store, 1, "A".into(), "A内容".into()).unwrap();

```

## 第 3 步：写单元测试 + 实现（GREEN）

> **核心思想**：从集成测试中提取需要的组件，先写单元测试，再写最小实现。

### 3.1 数据类型

```rust
#[derive(Serialize, Deserialize, Clone, Debug, PartialEq)]
pub struct Note {
    pub id: u32,
    pub title: String,
    pub content: String,
}
```

- `#[derive(...)]`：自动实现 trait
  - `Serialize`/`Deserialize`：serde 序列化（IPC 通信需要）
  - `Clone`：可复制（`store_load` 返回克隆值）
  - `Debug`：可打印调试信息
  - `PartialEq`：可比较（测试中 `assert_eq!` 需要）
- `pub`：公开字段（前端需要访问）

### 3.2 存储层

```rust
pub type NoteStore = Mutex<HashMap<u32, Note>>;

pub fn store_save(store: &mut HashMap<u32, Note>, note: Note) {
    store.insert(note.id, note);
}

pub fn store_load(store: &HashMap<u32, Note>, id: u32) -> Option<Note> {
    store.get(&id).cloned()
}

pub fn store_list(store: &HashMap<u32, Note>) -> Vec<Note> {
    store.values().cloned().collect()
}
```

- `type NoteStore = Mutex<HashMap<u32, Note>>`：类型别名，简化书写
- `Mutex`：互斥锁，保证多线程安全访问
- `HashMap<u32, Note>`：键值对存储，key 是笔记 ID
- `Option<Note>`：可能不存在（`get` 返回 `None`）
- `cloned()`：从引用克隆出值（避免借用问题）

### 3.3 单元测试

```rust
    #[test]
    fn test_store_save_and_load() {
        let mut store = HashMap::new();
        let note = Note { id: 1, title: "标题".into(), content: "内容".into() };
        store_save(&mut store, note.clone());
        assert_eq!(store_load(&store, 1).unwrap(), note);
    }

    #[test]
    fn test_store_load_nonexistent() {
        let store = HashMap::new();
        assert!(store_load(&store, 99).is_none());
    }

    #[test]
    fn test_store_overwrite() {
        let mut store = HashMap::new();
        store_save(&mut store, Note { id: 1, title: "旧".into(), content: "".into() });
        store_save(&mut store, Note { id: 1, title: "新".into(), content: "".into() });
        assert_eq!(store_load(&store, 1).unwrap().title, "新");
    }

    #[test]
    fn test_store_list() {
        let mut store = HashMap::new();
        store_save(&mut store, Note { id: 1, title: "a".into(), content: "".into() });
        store_save(&mut store, Note { id: 2, title: "b".into(), content: "".into() });
        assert_eq!(store_list(&store).len(), 2);
    }

    #[test]
    fn test_note_serialize() {
        let note = Note { id: 1, title: "t".into(), content: "c".into() };
        let json = serde_json::to_string(&note).unwrap();
        assert!(json.contains("\"id\":1"));
    }

    #[test]
    fn test_note_deserialize() {
        let json = r#"{"id":2,"title":"t","content":"c"}"#;
        let note: Note = serde_json::from_str(json).unwrap();
        assert_eq!(note.id, 2);
    }
```

### 3.4 命令层

```rust
fn cmd_save(store: &NoteStore, id: u32, title: String, content: String) -> Result<(), String> {
    let mut map = store.lock().map_err(|e| e.to_string())?;
    store_save(&mut map, Note { id, title, content });
    Ok(())
}
```

- `store.lock()`：获取 Mutex 锁
- `map_err(|e| e.to_string())`：将锁错误转为字符串
- `?`：错误时提前返回

### 3.5 Tauri 命令

```rust
#[tauri::command]
fn save_note(store: tauri::State<'_, NoteStore>, id: u32, title: String, content: String) -> Result<(), String> {
    cmd_save(&store, id, title, content)
}
```

- `#[tauri::command]`：标记为 Tauri 命令（前端可通过 `invoke` 调用）
- `tauri::State<'_, NoteStore>`：Tauri 注入的全局状态
- 命令名 `save_note` 对应前端 `invoke("save_note", ...)`

### 3.6 注册命令

```rust
pub fn run() {
    Builder::default()
        .manage(NoteStore::default())           // 注册全局状态
        .invoke_handler(tauri::generate_handler![  // 注册命令
            save_note, load_note, list_notes
        ])
        .run(tauri::generate_context!())
        .expect("error");
}
```

- `.manage()`：注册全局状态（Tauri 自动注入到 `tauri::State`）
- `generate_handler!`：宏，生成命令分发代码

## 第 4 步：连接前端

### 4.1 invoke 调用

```ts
import { invoke } from "@tauri-apps/api/core";

async function syncToRust(note: Note) {
  await invoke("save_note", { id: note.id, title: note.title, content: note.content });
}
```

- `invoke("save_note", { ... })`：调用 Rust 命令 `save_note`
- 参数名必须与 Rust 命令参数名一致（`id`、`title`、`content`）
- `await`：异步等待 Rust 返回

### 4.2 新建笔记

```ts
let nextId = 1;

async function createNote() {
  const id = nextId++;
  const note: Note = { id, title: "新笔记", content: "" };
  notes.push(note);
  switchNote(id);       // 先更新 UI
  await syncToRust(note); // 再同步到 Rust
}
```

- `nextId++`：递增 ID（不用 `Date.now()`，避免 u32 溢出）
- `switchNote` 放在 `syncToRust` 前面：**乐观更新**，UI 先响应，Rust 后台同步

### 4.3 编辑同步

```ts
editor.addEventListener("input", async () => {
  const note = notes.find((n) => n.id === activeNoteId);
  if (note) {
    note.content = editor.value;
    note.title = editor.value.split("\n")[0].replace(/^#+\s*/, "").trim() || "新笔记";
    await syncToRust(note);
    renderNoteList();
  }
  updateStats();
});
```

- 每次输入都同步到 Rust（后续版本可改为防抖）

### 4.4 初始化

```ts
async function init() {
  try {
    notes = await invoke<Note[]>("list_notes");  // 从 Rust 加载
  } catch {}
  if (notes.length === 0) {
    notes = [{ id: 1, title: "欢迎使用 NoteX", content: "# 欢迎使用 NoteX\n\n这是一个简单的记事本应用。" }];
    await syncToRust(notes[0]);  // 首次启动，写入默认笔记
  }
  nextId = Math.max(...notes.map((n) => n.id)) + 1;
  switchNote(notes[0].id);
}

init();
```

- `invoke<Note[]>("list_notes")`：调用 Rust 的 `list_notes` 命令
- `try/catch`：Rust 命令可能不存在（版本兼容）
- `nextId = max(id) + 1`：避免 ID 冲突
